提交代码到本地与远端

  1. git init //建立本地仓库
  2. 新建.ignore文件 //来忽略不需要提交的东西
  3. git add //指定需要提交到仓库的文件：git add
  4. git commit -m 第一次提交 //提交到本地仓库
  5. git remote add origin github地址 //关联远程仓库
  6. git push origin master //提交到远程仓库

.ignore文件无效

一开始没有添加.ignore文件，或者添加了后期又做了修改，.ignore文件会无效，原因是source tree默认是不会跟踪.ignore的变化的，必须清空source tree跟踪到的缓存信息，再重新add文件到本地仓库

git rm -r –cached .
git add .
git commit -m “Refresh adding .gitignore file.”

如果是中途从 .gitignore 文件中移除某一文件类型，想要这个文件类型重新被 track，需要通过以下命令行的方式，让 .gitignore 文件生效：

git add -f *.class
git commit -m “Refresh removing .class from .gitignore file.”

git pull//与GitHub远程仓库同步

出现Permission denied (publickey) 错误

本地端的SSH key与远端仓库的SSH不一致导致的，本地生成一个key，在添加到远端即可后面连续回车就好了，生成的文件会保存在/Users/you/.ssh/id_rsa中，打开它复制里面的全部文本到Github的新的SSH key 即可
ssh-keygen -t rsa -b 4096 -C wenmingvs@gmail.com
